The process of evolving the web through web standards has, for a long time now, relied heavily on cross-browser test suites. When creating a new feature specification, writing tests and experimental implementations go hand in hand. A single, shared test suite helps browsers make sure new features are interoperable, and helps users of the web platform track the implementation status.

But what these shared test suites could not tell you, until very recently, is whether any web feature has interoperable or complete accessibility support. Accessibility support defines how a web feature is consumed and interacted with by a user who cannot use a screen, mouse, or keyboard. This last year we have made great strides in expanding the testability of the accessibility of web features within WPT and WebDriver. In this talk I'll discuss the tests you can now write, the challenges we had getting here, and the status of work still in progress.
